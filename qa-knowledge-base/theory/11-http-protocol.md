# 11. Протокол HTTP

**HTTP (HyperText Transfer Protocol)** — протокол передачи данных, работающий по модели «запрос-ответ». 

## 1. Ключевые особенности
* **Stateless (Без сохранения состояния):** Сервер не запоминает клиента между запросами. Каждый запрос самодостаточен (для обхода этого используют Cookie и Токены).
* **Работает поверх TCP/IP:** Гарантирует доставку данных в правильном порядке.
* **HTTPS:** Расширение протокола с шифрованием (SSL/TLS) для безопасной передачи данных.



## 2. Структура HTTP-запроса (Request)
Когда ты нажимаешь кнопку в браузере, улетает сообщение:
1.  **Метод (Method):** Что нужно сделать?
2.  **URL (Путь):** К какому ресурсу обращаемся?
3.  **Заголовки (Headers):** Метаданные (тип данных, авторизация, информация о браузере).
4.  **Тело (Body):** Сами данные (например, логин и пароль в формате JSON). Используется в POST/PUT/PATCH.

### Основные методы:
* **GET:** Получение данных (тела обычно нет).
* **POST:** Создание новой сущности (данные в теле).
* **PUT:** Полное обновление существующей сущности.
* **PATCH:** Частичное обновление.
* **DELETE:** Удаление.

## 3. Структура HTTP-ответа (Response)
Сервер обрабатывает запрос и возвращает:
1.  **Код состояния (Status Code):** Результат обработки.
2.  **Заголовки (Headers):** Тип контента, дата, серверные настройки.
3.  **Тело (Body):** Данные, которые вернул сервер (HTML-страница, JSON с товарами и т.д.).

## 4. Коды состояний (Status Codes)
Их нужно знать наизусть по группам:
* **1xx (Informational):** Информация (редко встречаются в тестировании).
* **2xx (Success):** Всё хорошо.
    * `200 OK` — Успех.
    * `201 Created` — Объект успешно создан.
* **3xx (Redirection):** Перенаправление.
    * `301 Moved Permanently` — Ресурс переехал навсегда.
* **4xx (Client Error):** Ошибка клиента (ты отправил что-то не то).
    * `400 Bad Request` — Ошибка в синтаксисе запроса.
    * `401 Unauthorized` — Не авторизован (нужен логин).
    * `403 Forbidden` — Доступ запрещен (нет прав).
    * `404 Not Found` — Ресурс не найден.
* **5xx (Server Error):** Ошибка сервера (баг в коде бэкенда).
    * `500 Internal Server Error` — Что-то сломалось на сервере.
    * `502 Bad Gateway` — Проблема со связью между серверами.
    * `503 Service Unavailable` — Сервер перегружен или на обслуживании.



## 5. Параметры запроса
* **Query Params:** Передаются прямо в URL после знака `?`. 
    * *Пример:* `wildberries.kz/search?query=iphone15`.
* **Path Params:** Часть самого пути. 
    * *Пример:* `api/users/123` (где 123 — ID пользователя).

### Зачем это тестировщику?
Если кнопка «Войти» не работает:
1. Открываешь **Network** в DevTools.
2. Смотришь код ответа.
3. Если **4xx** — скорее всего, фронтенд шлет неверные данные.
4. Если **5xx** — бежим к бэкенд-разработчику, упал сервер.
