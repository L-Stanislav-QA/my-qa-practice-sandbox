# 22. Инструменты повышения качества: Mocking и Overrides

Использование расширенных функций DevTools позволяет имитировать редкие условия, проверять гипотезы безопасности и анализировать производительность без внесения изменений в исходный код проекта.

## 1. Мокирование сетевых запросов
Имитация ответов сервера необходима для проверки стабильности фронтенда и корректности обработки исключений.

### Сценарии применения:
* **Обработка статус-кодов:** Подмена ответа сервера на `401 Unauthorized` (проверка редиректа на логин), `403 Forbidden` (проверка прав доступа) или `500 Internal Server Error` (проверка отображения страницы ошибки).
* **Пустые состояния (Empty States):** Подмена JSON-ответа на пустой список для проверки наличия заглушек в интерфейсе (например, надписи «У вас пока нет заказов»).
* **Валидация типов данных:** Замена ожидаемого числа в JSON на строку или `null` для проверки устойчивости фронтенд-логики к некорректным данным.

### Способы реализации:
1. **Блокировка ресурсов:** Использование «Block Request URL» для проверки поведения сайта при потере критических файлов (шрифтов, скриптов аналитики, стилей).
2. **Network Response Mocking:** Использование вкладки Network для создания моков на лету (в современных версиях Chrome доступно через контекстное меню «Override content»).

## 2. Local Overrides (Локальное переопределение)
Механизм подмены серверных файлов локальными копиями. Это позволяет тестировщику выступать в роли «квази-разработчика» для верификации багов.

### Сценарии применения:
* **Инъекция тестовых скриптов:** Добавление console.log() в JS-файлы приложения для отслеживания промежуточных значений переменных без пересборки проекта.
* **Проверка гипотез верстки:** Массовое изменение CSS-классов для проверки адаптивности на всех страницах сайта одновременно.
* **Тестирование безопасности:** Отключение атрибутов `disabled` у кнопок или удаление JS-функций валидации перед отправкой формы для проверки защищенности бэкенда.

### Технический процесс:
1. Выбор папки в **DevTools -> Sources -> Overrides**.
2. Изменение файла во вкладке **Elements** (для CSS) или **Sources** (для JS).
3. Сохранение изменений (`Ctrl+S`). После перезагрузки страницы в углу значка файла в Network появится иконка фиолетового кружка, сигнализирующая о подмене.

## 3. Инструменты отладки и анализа (Console & Rendering)
Помимо логирования, консоль и дополнительные панели предоставляют инструменты для глубокой инспекции.

* **Console API:** Использование команд `console.table()` для удобного просмотра больших JSON-массивов или `console.time()` / `console.timeEnd()` для замера скорости выполнения функций.
* **Rendering Tab:** Позволяет визуализировать «перерисовки» (Paint flashing) — это помогает найти лишние нагрузки на браузер, которые вызывают тормоза интерфейса.
* **Emulate Vision Deficiencies:** Эмуляция различных типов нарушений зрения (дальтонизм) для проверки доступности (Accessibility) интерфейса.

## 4. Throttling и эмуляция сенсоров
Проверка продукта в условиях ограниченных аппаратных и сетевых ресурсов.

### Эмуляция сети и процессора:
* **Network Throttling:** Создание профилей с кастомной задержкой (Latency) для имитации спутникового интернета или нестабильного Wi-Fi.
* **CPU Throttling:** Ограничение мощности процессора (в 4 или 6 раз) для выявления «тяжелых» анимаций и скриптов, блокирующих основной поток (Main Thread).

### Эмуляция устройств и окружения (Панель Sensors):
* **Geolocation:** Подмена координат для тестирования сервисов, работающих с картами или региональными ценами.
* **Touch-эмуляция:** Проверка того, как элементы реагируют на касания (touch-события) вместо кликов мышью.
* **Idle Detector:** Проверка поведения сайта при переходе системы в спящий режим.

## 5. Lighthouse и Web Vitals
Автоматизированный аудит, фокусирующийся на пользовательском опыте (UX).

* **LCP (Largest Contentful Paint):** Время загрузки самого большого видимого элемента.
* **FID (First Input Delay):** Время до момента, когда пользователь может начать взаимодействовать со страницей.
* **CLS (Cumulative Layout Shift):** Измерение «прыжков» верстки при загрузке (когда текст уезжает вниз из-за подгрузившейся картинки).
