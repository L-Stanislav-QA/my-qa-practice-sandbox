# 15. Работа в Postman: От запросов к автотестам

**Postman** — основной инструмент для тестирования API. Он позволяет отправлять запросы, организовывать их в сценарии и автоматизировать проверки.

## 1. Организация: Collections и Folders
* **Collections (Коллекции):** Группа запросов, объединенных одной темой (например, проект «Склад»). Позволяет настраивать авторизацию сразу для всех вложенных запросов.
* **Folders (Папки):** Разделение внутри коллекции по функционалу (например, «Авторизация», «Корзина», «Негативные тесты»).



## 2. Переменные и Окружения
Использование переменных избавляет от необходимости вручную менять данные при смене сервера.
* **Environments (Окружения):** Наборы данных для разных стендов (Dev, Stage, Prod). 
* **Синтаксис:** Переменная вставляется через двойные фигурные скобки: `{{base_url}}`.
* **Динамические переменные:** Генерация случайных данных для тестов: `{{$randomEmail}}`, `{{$randomInt}}`, `{{$randomUserName}}`.

## 3. Скрипты и Автоматизация
Postman поддерживает написание кода на JavaScript в двух вкладках:

### Pre-request Script
Выполняется **ДО** отправки запроса. Используется для подготовки данных: например, вычисление текущей даты или создание хэша пароля.

### Tests
Выполняется **ПОСЛЕ** получения ответа. Здесь пишутся проверки (assertions).

**Примеры тестов:**

* Проверка кода ответа на 200 OK:
```json
pm.test("Status code is 200", function () {
pm.response.to.have.status(200);
});
```
* Проверка времени ответа (не более 400 мс):
```json
pm.test("Response time is less than 400ms", function () {
pm.expect(pm.response.responseTime).to.be.below(400);
});
```
* Проверка значения в JSON-теле ответа:
```json
pm.test("Check user role is admin", function () {
var jsonData = pm.response.json();
pm.expect(jsonData.role).to.eql("admin");
});
```
## 4. Передача данных (Chaining)
Позволяет автоматически забирать данные из одного ответа и подставлять в другой запрос. 
* **Сценарий:** Запрос «Login» получает токен. Вкладка **Tests** сохраняет его: `pm.environment.set("token", jsonData.token);`. 

Следующий запрос на «Профиль» уже использует этот токен в заголовках.

## 5. Запуск коллекций (Runner)
Инструмент для автоматического прогона всех запросов в папке по очереди. Позволяет задать количество итераций и задержку между запросами. В конце выдает отчет: сколько тестов (assertions) прошли успешно, а сколько упало.

---

### Что проверять QA в Postman:
1. **Статус-коды:** Соответствуют ли они документации?
2. **Структура JSON:** Все ли поля присутствуют и имеют правильный тип данных?
3. **Обработка ошибок:** Что вернет API, если отправить пустые обязательные поля?
4. **Безопасность:** Работает ли запрос без токена авторизации?
